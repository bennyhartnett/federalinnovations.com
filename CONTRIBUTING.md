# Contributing to Federal Innovations

Thank you for your interest in contributing to Federal Innovations! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check existing [issues](../../issues) to avoid duplicates
2. Open a new issue using the appropriate template
3. Provide as much detail as possible

### Making Changes

1. **Fork the repository** and create your branch from `main`
2. **Make your changes** following our guidelines below
3. **Test your changes** locally by opening `index.html` in a browser
4. **Commit your changes** with clear, descriptive messages
5. **Open a Pull Request** with a description of your changes

### Code Guidelines

- Keep the codebase simple and maintainable
- Ensure responsive design works on mobile devices
- Test across multiple browsers when possible
- Follow existing code style and formatting

### Commit Messages

Write clear and meaningful commit messages:

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Keep the first line under 72 characters
- Reference issues when applicable

### Pull Request Process

1. Update documentation if needed
2. Ensure your changes don't break existing functionality
3. Request review from maintainers
4. Address any feedback promptly

## Code of Conduct

Please note that this project follows our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code.

## Questions?

Feel free to reach out at [contact@federalinnovations.com](mailto:contact@federalinnovations.com) if you have any questions.

Thank you for contributing!
